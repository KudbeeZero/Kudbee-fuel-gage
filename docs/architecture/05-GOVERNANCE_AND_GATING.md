# 05 — Governance and Gating Architecture

## Overview

Kudbee's governance plane is a multi-layered enforcement system that gates every agent action through cryptographic contracts, receptor-ligand affinity binding, and human-in-the-loop (HITL) approval gates. The HERMES auditor provides continuous oversight, while the probation docket tracks agent misbehavior.

---

## Receptor Gating Engine

**Implementation:** `services/memory/src/receptorGating.ts`

The Receptor Gating Engine models AI agent-token interactions as **biophysical receptor-ligand binding events** across a 3D coordinate lattice. Tokens are evaluated against cell-slot coordinates using weighted multi-dimensional affinity scoring.

### Core Concepts

- **Cell Slot Coordinates:** Each slot is a point in 3D space `(x, y, z)` — representing a semantic position in the architecture
- **Token Embedding:** 1536-dimension vectors generated by Gemini `text-embedding` (or local deterministic fallback)
- **Binding Affinity (Kd):** Dissociation constant modeling receptor-ligand binding strength. `Kd → 0` indicates near-perfect binding.
- **Efficacy (ε):** Intrinsic activity weight (0.0–1.0). A token may bind to a slot but have limited mutation power (ε close to 0).

### Affinity Scoring

The engine evaluates admission via `receptorGate.evaluateAdmission()`:

```typescript
{
  tokenId, tokenHash, embedding, kd, efficacy, slot, tokenType
}
```

Token types: `ORDINARY`, `CHALLENGE_TOKEN`, `ADMIN`

**Gating formula:**

$$\\phi_r = \\frac{\\|\\mathbf{x}(t)\\|_2}{\\|\\mathbf{x}(t)\\|_2 + K_d}$$

Where `φr` is the binding probability/occupancy, `||x(t)||_2` is the token's normalized similarity score, and `Kd` is the dissociation constant.

### Lock and Shield (Suboxone Effect)

Critical cell-slot coordinates can be occupied by **high-affinity, zero-efficacy Guard Tokens** (`ε → 0`, `Kd → 0`) that physically crowd out ordinary tokens. Once locked, a slot requires dual-signature cryptographic override from the Governance Gate to unlock.

**Fields in `think_tokens`:**
- `locked_by` — Guard token hash occupying the coordinate slot
- `kd` — Gating affinity threshold
- `efficacy` — Intrinsic activity weight

## AGC Contracts (Agent Governance Contracts)

**Implementation:** `services/lib/agcContract.ts`

AGC Contracts are **Assume-Guarantee contracts** — enforceable policy definitions for agent behavior. Each contract defines numeric bounds that the kernel verifies against every token produced by the agent.

### AGCSchema (Zod)

```typescript
{
  maxTokensPerWindow: number,   // Token budget per time window
  maxMemoryBytes: number,       // Memory allocation limit
  maxLatencyMs: number,         // Maximum operation latency
  minSimilarityScore: number,   // Minimum semantic similarity threshold
  // ... additional contract fields
}
```

### Contract Lifecycle

1. **Sign** — `POST /api/governance/contract/sign` — validates via `AGCSchema.safeParse()`, persists to Redis `kudbee:contract:*`
2. **Verify** — `POST /api/governance/contract/verify/:id` — `verifyContract(id, token)` checks token compliance
3. **List Active** — `GET /api/governance/contract/active` — returns all active contracts via `getActiveContracts()`
4. **Contract State:** `ContractState` enum tracking contract lifecycle

### Endpoints

| Method | Route | Description |
|:---|:---|:---|
| `POST` | `/api/governance/contract/sign` | Sign a new AGC contract |
| `POST` | `/api/governance/contract/verify/:id` | Verify token against contract bounds |
| `GET` | `/api/governance/contract/active` | List all active contracts |

## Think Token Lifecycle

**Implementation:** `services/memory/thinkTokenGenerator.ts`, `packages/types/index.ts`

Think Tokens are permanent correction deltas — semantic memory artifacts that record a failure state, the correction applied, and the reasoning context.

### Token States

```
candidate → admitted ⟷ locked → challenged → evicted | archived
```

Canonical lifecycle: `PENDING_APPROVAL` → `VERIFIED` → `RECYCLED` | remains `PROVEN`

### ThinkTokenSchema

| Field | Type | Description |
|:---|:---|:---|
| `id` | `string` | UUID primary key |
| `original_trace_id` | `string?` | Source telemetry trace |
| `task_context` | `Record<string, unknown>?` | Original task data |
| `failed_state` | `Record<string, unknown>?` | Pre-correction state |
| `correction_delta` | `string` | Correction applied |
| `embedding` | `number[]` | 1536-dim vector |
| `status` | `PENDING_APPROVAL \| VERIFIED \| RECYCLED \| PROVEN` | Token lifecycle |
| `token_cost` | `number` | Compute cost |
| `kd` | `number` | Dissociation constant |
| `efficacy` | `number` | Activity weight (0.0–1.0) |
| `locked_by` | `string \| null` | Guard token hash |

### Uncertainty-Proxy Confidence Scores

The `/api/think/trajectories` endpoint derives a `confidence_score` for each token:

| Status | Derived Confidence | Gating Result |
|:---|:---|:---|
| `VERIFIED` | 0.95 | Clear the gate |
| `PROVEN` | 0.55 | Ambiguous |
| `RECYCLED` | 0.25 | Trapped by gate |

Explicit `confidence_score` in `task_context` overrides the derived value.

### Endpoints

| Method | Route | Description |
|:---|:---|:---|
| `POST` | `/api/governance/mint-think-token` | Mint a new think token with optional receptor gating |
| `GET` | `/api/think/trajectories` | List trajectories with confidence scoring |
| `PATCH` | `/api/think/trajectories/:hash/status` | Update token status (VERIFIED/RECYCLED) |
| `POST` | `/api/think/archive` | Archive chain-of-thought reasoning |
| `GET` | `/api/think/archive` | Retrieve recent thought blocks |
| `POST` | `/api/think/synthesize` | Groq LPU synthesis of think tokens |
| `GET` | `/api/think/metrics` | Aggregated think token metrics |
| `GET` | `/api/think/anomalies` | Low-confidence anomaly tokens |

| `GET` | `/api/memory/think-tokens` | pgvector similarity search for past successes |

**Implementation:** `services/lib/probationRegistry.ts`

### Stages

1. **Pending** — Agent flagged; docket entry created in `kudbee:probation:pending` (sorted set)
2. **Evaluation** — HERMES auditor reviews; deadline-based via sorted set scores

## HERMES Auditor

**Implementation:** `services/agents/hermes.js`

HERMES is the continuous oversight agent — it audits AGC contract compliance, monitors agent behavior, and records structured reasoning to the `think` table.

### Capabilities

- `archive_thought(thought_block)` — Archival with Resilient-First persistence
- `query_system_topology(query, limit)` — Blueprint-first semantic search (mandated by system prompt)
- **Heartbeat:** Posts to `POST /api/health` every ~15s; stored in `kudbee:agents:hermes` with 30s TTL
- **Staleness:** Treated as Offline if heartbeat > 45s old

### Blueprint-First Rule

Before writing routes, modifying schemas, or changing infrastructure, HERMES MUST call `query_system_topology` to inspect the verified architecture.

### Endpoints

| Method | Route | Description |
|:---|:---|:---|
| `GET` | `/api/governance/health` | Governance health (HERMES status + proposed count) |
| `GET` | `/api/governance/hermes-logs` | Last 50 HERMES auditor log lines |
| `POST` | `/api/health` | HERMES heartbeat sink |

## HITL Governance Gate

**Implementation:** `services/governance/router.js`, inline in `server.js`

Human-in-the-Loop approval gate for proposed agent actions. The dashboard surfaces a "Governance Intervention Required" card when actions await approval.

### Flow

1. **Propose** — Action enters `PROPOSED` state via `proposeAction()`
2. **Pending List** — `GET /api/governance/pending` returns `ApprovalRequest[]` for the dashboard
3. **Resolve** — `POST /api/governance/resolve` with `{ id, decision: 'APPROVE' | 'REJECT' }`
4. **Broadcast** — SSE event fired on approve/reject

### Endpoints

| Method | Route | Description |
|:---|:---|:---|
| `GET` | `/api/governance/pending` | List pending approval requests |
| `POST` | `/api/governance/resolve` | Approve or reject a proposed action |
| `POST` | `/api/governance/approve` | Approve by ID |
| `POST` | `/api/governance/reject` | Reject by ID |

## Probabilistic Uncertainty Gating

**Implementation:** `services/agents/router.ts` → `routeAgentPayload()`

The hard anti-hallucination circuit. Every agent payload carries `confidence_score` (0.0–1.0) and `uncertainty_flag` (boolean).

### Threshold

- `UNCERTAINTY_THRESHOLD = 0.8`
- If `confidence_score < 0.80` OR `uncertainty_flag === true` → route to `PENDING_APPROVAL` tagged `REASON: HIGH_UNCERTAINTY`
- Resilient-First: never throws; a sink failure still returns `PENDING_APPROVAL`

### Endpoint

| Method | Route | Description |
|:---|:---|:---|
| `POST` | `/api/agents/evaluate` | Full uncertainty gate evaluation + routing |
| `GET` | `/api/agents/evaluate` | Parse-only variant for diagnostics |
